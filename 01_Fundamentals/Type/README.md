# Type（型）

## 概要
データの表現・制約・メモリ管理・振る舞いを決定するもの

---

## 1. struct（値型）
### 特徴
- スタック(スコープ終了時に廃棄)
- コピーされる
- 軽量・高速

### 基本型（Primitive Types）
言語標準で定義されている、データの最小単位となる型です。

| 分類 | 型名 | 用途・特徴 |
| :--- | :--- | :--- |
| **整数** | `int`, `long`, `byte` | ループのカウント、ID、バイナリデータ |
| **浮動小数点** | `double`, `float` | 科学計算、ゲームの座標（計算速度重視） |
| **高精度小数** | `decimal` | **金額計算、税計算**（誤差を許容しない） |
| **論理** | `bool` | `true` / `false` の条件判定 |
| **文字** | `char` | 1文字（Unicode）の表現 |

### 列挙型（enum）
「状態」や「種類」など、あらかじめ決められた選択肢に名前をつけて管理する型です。  
| 分類 | **役割** |
| :--- | :--- |
| 整数 | 意味のない数値（マジックナンバー）を排除し、コードの意図を明確にする。 |

### 構造体型（struct）
複数のデータを1つにまとめた「軽量な値の塊」です。
| 分類 | **例** |
| :--- | :--- |
| 標準の構造体 | DateTime (日時), TimeSpan (時間差), Guid (識別子)
| カスタム構造体 | public struct Point { public int X, Y; }

---

## 2.　class（参照型）
- ヒープ(GCによって管理)
- 参照を共有
- 柔軟

---

## 3. 型安全性

型により不正なデータを防ぐ

例：
- intに文字列は入らない

---

## 4. 静的型付け vs 動的型付け

### 静的型付け（C#）
- コンパイル時に型確定
- 安全・高速

### 動的型付け
- 実行時に型決定
- 柔軟だが不安定

---

## 5. 数値型の選択

### int / long
- 高速
- 整数

### double
- 高速だが誤差あり

### decimal
- 遅いが高精度（金融向け）

---

## 6. nullとNullable

### null
参照型で使用

### Nullable<T>
値型でもnullを扱う

例：
int? age = null;

---

## 7. ジェネリック型

型を抽象化する仕組み

例：
List<T>

メリット：
- 型安全
- 再利用性

---

## 8. 設計としての型

型は単なるデータではない

- 意味を表す
- 制約を与える
- バグを防ぐ

---

## 9. パフォーマンス観点

- 値型：高速（ただしコピーコストあり）
- 参照型：柔軟（GCコストあり）

---

## 10. 結論

型とは

- メモリ構造
- 安全性
- 設計

を決定するもの
